Unit 05

L’area riservata dello studio. La password non è più scritta nel file HTML: viene verificata da una funzione sul server Supabase.

Progetto
Supabase — convkywqutizvkwmgcda.supabase.co
Login

Sicurezza

Come viaggia una richiesta

Due percorsi diversi partono dallo stesso dominio, a seconda di cosa sta facendo il visitatore.

PERCORSO — Sito pubblico
  1. Il browser cerca diaprod.it
  2. Cloudflare risolve il DNS e fa da proxy
  3. La richiesta arriva a GitHub Pages
  4. Se qualcuno si iscrive, il form chiama il Worker
  5. Il Worker inoltra a MailerLite
PERCORSO — Report Giornaliero
  1. Si apre la pagina del report (HTML)
  2. Nome + password vengono inviati a Supabase
  3. Supabase verifica la password lato server
  4. Se corretta, si accede e l’evento viene loggato
  5. Al logout, un secondo evento viene loggato
PERCORSO — Email in arrivo
  1. Un’email arriva per studio@diaprod.it
  2. Il record MX (DNS only) punta a Register.it
  3. SPF/DKIM/DMARC autenticano il mittente
  4. Register.it consegna nella mailbox
  5. Accessibile via IMAP/webmail o client di posta
